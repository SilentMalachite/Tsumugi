# ADR 0028: 令和6年度 就労継続支援B型 主要加算の実値（送迎・欠席時対応・食事提供体制・目標工賃達成指導員配置・福祉専門職員配置等・初期・処遇改善）

## 結論

令和6年度改定の就労継続支援B型の主要加算候補7家族（spec §3.1-3）のうち、固定単位加算6家族（福祉専門職員配置等加算(Ⅰ)(Ⅱ)(Ⅲ)、初期加算、欠席時対応加算、目標工賃達成指導員配置加算、食事提供体制加算、送迎加算(Ⅰ)(Ⅱ)＋同一敷地variant）の全15サービスコード行と、%ベースの処遇改善加算系2世代（令和6年4月〜5月の旧3制度＝処遇改善(Ⅰ)〜(Ⅲ)・特定処遇改善(Ⅰ)(Ⅱ)・ベースアップ等支援、令和6年6月〜令和8年5月の統一 福祉・介護職員等処遇改善加算(Ⅰ)〜(Ⅳ)）の全率を、ハッシュ照合済み一次資料から確定した。固定単位行は `r6-service-codes-2-xlsx` と `r6-service-codes-2-pdf` の独立2抽出が全行一致し、告示改正行（食事提供体制・目標工賃達成指導員配置・送迎注1）は `r6-fee-notice` とも一致した。R6改正表に現れない不変更行（福祉専門職員・初期・欠席時対応・送迎単位数）は、ハッシュ登録済みの現行統合本文 `current-fee-notice-html` の同一値とR6/R8コード表の同一コード・同一単位数によるcontinuity mappingで確定した。処遇改善系の率は `r6-fee-notice` の第1条表（4月施行）と第2条表（6月施行）から2方式（`pdftotext -layout` / `-raw`）で独立抽出し全率一致した。適用順（CalculationOrder）と丸めはADR 0025の `MonthlyTargetUnitSum` 契約で確定できたため、処遇改善系を決定表に含める。確定できなかったのは、送迎加算の「別に厚生労働大臣が定める送迎」の要件本文、R8-06の処遇改善新率・新コード対応、食事提供体制加算の附則経過措置期限の3点であり、`docs/open-questions.md` に起票した。本ADRの値は Task 11 Step 2 の `additions.json` seedとgolden case期待値の唯一の出典である。

- ハード制約3（CLAUDE.md）は単位数・加算のハードコードを禁止する。ADR 0020が出典（documentId・sha256）と版束（`claim-master-r6-04`〜`claim-master-r8-06`）、ADR 0021が加算ごとの請求コード群・OfficeCapability正式キー・必要入力、ADR 0025が割合加減算のsource row契約（`percentageBaseScope` / `targetSelector` / `calculationOrder` / 丸め）、ADR 0027がB型基本報酬135行と地域区分単価を確定済み。本ADRは加算の**実値**（単位数・率・算定単位・丸め・適用順）を確定する。

## 選択肢

### A: コード表2形式の二重照合＋告示改正値＋現行統合本文continuityで候補7家族を採録する（採用）

- 固定単位行はxlsxとPDFの独立2ファイルで突合し、転記・読取り事故を機械的に検出する。実施の結果、対象15行の全てで一致した。
- R6改正告示に現れない不変更行も、ハッシュ登録済み現行統合本文の同一値とR6/R8コード表の同一コード・同一単位数で一意に確定でき、推測を排除できる。

### B: コード表Excel単独から抽出する

- 告示・PDF・現行本文との突合がなく、合成単位数の誤り・算定単位の欠落（初期加算の49列目は空欄）を検出できない。不採用。

### C: B型加算26コードグループ全てを一括採録する

- ADR 0021の26グループのうちTask 11候補外（就労移行支援体制・医療連携・重度者支援体制等）は請求計算の実装順が後であり、本ADRで値を確定しても検証負荷だけが増える。候補7家族に限定する。不採用。

## 決定

### 1. 適用版と判定原則

- 固定単位加算（決定3）は `claim-master-r6-04`（2024-04-01）から有効。R8-6月表でも同一コード・同一単位数を照合済みのため、`claim-master-r8-06` まで同値で継続する（廃止月null）。
- 処遇改善系（決定4）は世代で分かれる: 旧3制度は2024-04-01〜2024-05-31（`claim-master-r6-04` のみ）、統一(Ⅰ)〜(Ⅳ)は2024-06-01〜2026-05-31（`claim-master-r6-06`〜`claim-master-r7-09`）。**R8-06の処遇改善は率・コード構成が変わるため本ADRのスコープ外**（決定7・open-questions）。

補足（抽出上の確定根拠）:

- **continuity mappingReason（不変更行）**: 福祉専門職員(15/10/6)・初期(30)・欠席時対応(94・月4回)・送迎(21/10・70%)はR6改正告示の対照表に単位数が現れない（`r6-fee-notice` p.149で送迎のイ・ロは「（略）」）。現行統合本文 `current-fee-notice-html`（2026-06時点）の第14の該当項が同一値を条文で定め、かつR6コード表（xlsx・PDF）とR8コード表が同一コード・同一単位数であることを全行照合した。R6〜R8で値の変更が挟まれていないことはこのコード表同値で裏付ける。
- **初期加算の算定単位**: `r6-service-codes-2-xlsx` 937行の49列目（算定単位）は空欄であり、xlsx/PDF単独では算定単位を確定できない。現行第14の4注「利用を開始した日から起算して30日以内の期間について、1日につき所定単位数を加算する」で確定した（xlsx算定項目列の「初期加算（利用開始日から３０日を限度）」と整合）。
- **同一敷地内70%の丸め**: xlsxの合成単位数は `ROUND(21×0.7)=15`・`ROUND(10×0.7)=7`（四捨五入）で、ADR 0025の `claim.rounding.units.half-up.v1` と一致する。同一敷地variantは合成済み固定単位の実在コードとしてseedし、runtimeで70%を再計算しない。
- **欠席時対応加算の条件**: 利用予定日の急病等による利用中止＋従業者による利用者・家族等との連絡調整その他の相談援助＋状況・内容の記録、1月につき4回を限度（現行第14の9注・`r6-calculation-note` p.297）。
- **算定条件（OfficeCapability届出）**: ADR 0021の正式キーに従う — 福祉専門職員=`welfare-professional-staffing`のⅠ/Ⅱ/Ⅲ選択（Ⅰ>Ⅱ>Ⅲの排他は現行第14の8注2・注3）、目標工賃達成指導員=`target-wage-instructor.2`、食事提供体制=`meal-provision-system.2`＋受給者証上の対象＋当日提供実績、送迎=`transport-system`の3(Ⅰ)/4(Ⅱ)。初期加算・欠席時対応加算は体制届出不要で実績のみから算定する（ADR 0021の必要入力欄）。

### 4. %ベース処遇改善系（月次割合加算）

ADR 0025の割合加減算source row契約に従い、全行とも `PercentageOfTargetAmount` / `percentageBaseScope = MonthlyTargetUnitSum` / `percentageApplicationKind = Add` / `targetSelector = target.b46.items-1-to-16-4.v1` / `calculationStepId = claim.step.units.monthly-target.percentage.v1` / `roundingRuleId = claim.rounding.units.half-up.v1` / 算定単位「1月につき」とする。基底（1から16の4までにより算定した単位数）は告示条文で固定されるため%行同士は複利にならない。率は浮動小数を使わず `decimal`（1000分のnのままn/1000）で保持する。

**適用順（CalculationOrder）の確定**: 統一(Ⅰ)〜(Ⅳ)は相互排他（告示ただし書き）で同時算定がなく、order=1。旧3制度は各制度が同一基底「1から16の4までにより算定した単位数」に独立に加算されるため（告示第14の17・18・19の各注）、算定結果は適用順に依存しない。ADR 0025がorderの穴・重複を拒否するため、告示の条番号順に 処遇改善=1、特定処遇改善=2、ベースアップ等支援=3 を固定する。各制度グループ内の排他（処遇改善Ⅰ〜Ⅲ、特定Ⅰ/Ⅱ）は告示ただし書きどおり。

### 6. 手計算検証ケース（golden case期待値）

端数規則はADR 0025に従う: 固定単位加算は整数単位×回数（丸めなし）、%行は月次対象合計×率を `claim.rounding.units.half-up.v1` で整数化して加算、総費用額＝最終給付単位数×地域単価の円未満切捨て、1割相当額＝総費用額×10/100の円未満切捨て、給付費＝総費用額−決定利用者負担額。3ケースとも法31条特例不適用・上限は1割相当額以上・上限額管理対象外を前提とする（ADR 0027決定4と同一前提）。

#### ケースA: ADR 0027ケース1＋固定単位加算4種（cap-20-or-less × band-20000-25000 × staff-7.5-1 × region-grade-2）

- 基本: 462049（就継ＢⅡ１５）637単位×22日 = 14,014単位
- 福祉専門職員配置等加算(Ⅰ): 466037、15単位×22日 = 330単位
- 食事提供体制加算: 465070、30単位×20日（食事提供20日）= 600単位
- 送迎加算(Ⅰ): 466590、21単位×40片道（20日往復）= 840単位
- 欠席時対応加算: 466040、94単位×2回 = 188単位
- 月次給付単位数: 14,014 + 330 + 600 + 840 + 188 = **15,972単位**
- 総費用額: 15,972 × 10.91円 = 174,254.52円 → 円未満切捨て → **174,254円**
- 1割相当額: 174,254 × 10/100 = 17,425.4円 → 円未満切捨て → 17,425円
- 給付費: 174,254 − 17,425 = **156,829円**

#### ケースB: ケースA＋統一処遇改善加算(Ⅰ)（2024-06以降の月）

- 月次対象単位合計（`target.b46.items-1-to-16-4.v1`）: 15,972単位（ケースAの全行が対象。%行自身は含まない）
- 処遇改善(Ⅰ): 15,972 × 93/1000 = 1,485.396 → `claim.rounding.units.half-up.v1` → **1,485単位**（`claim.step.units.monthly-target.percentage.v1`）
- 最終給付単位数: 15,972 + 1,485 = **17,457単位**
- 総費用額: 17,457 × 10.91円 = 190,455.87円 → 円未満切捨て → **190,455円**
- 1割相当額: 190,455 × 10/100 = 19,045.5円 → 円未満切捨て → 19,045円
- 給付費: 190,455 − 19,045 = **171,410円**

#### ケースC: ADR 0027ケース2＋定員連動・初期・同一敷地送迎（cap-20-or-less × band-under-10000 × staff-10-1 × region-other）

- 基本: 462883（就継ＢⅢ１８）490単位×20日 = 9,800単位
- 目標工賃達成指導員配置加算（定員20人以下）: 465255、45単位×20日 = 900単位
- 初期加算: 465050、30単位×10日（利用開始日から30日以内のサービス提供日10日）= 300単位
- 送迎加算(Ⅱ)（同一敷地内）: 466593、7単位×36片道（18日往復）= 252単位
- 月次給付単位数: 9,800 + 900 + 300 + 252 = **11,252単位**
- 総費用額: 11,252 × 10.00円 = 112,520.00円 → 円未満切捨て → **112,520円**
- 1割相当額: 112,520 × 10/100 = 11,252.0円 → 円未満切捨て → 11,252円
- 給付費: 112,520 − 11,252 = **101,268円**

## 影響

- Task 11 Step 2のseed（`additions.json`）は決定3・4の行と決定2のトークンだけを使う。ADRにない値・キー・selectorをseedに追加する場合は本ADRの改訂又は新ADRを先行させる。
- `ClaimCalculator` の加算明細は、固定単位行を `UnitsPerCountAmount`（`claim.step.units.multiply-count.v1`・丸めなし）、%行を `PercentageOfTargetAmount`（`MonthlyTargetUnitSum`・`claim.step.units.monthly-target.percentage.v1`・`claim.rounding.units.half-up.v1`）で実装する。決定6の3ケースをgolden caseとして固定する。
- 決定5のgap 3点（同一敷地内送迎の判別、利用開始日、欠席時対応の構造化記録）は、実装dispatchがseedで補わずreadiness gate／入力UIで顕在化させる。
- 再検証手順: `sources.json` の各URLを取得し `shasum -a 256` が登録値と一致することを確認した上で、(1) `r6-service-codes-2-xlsx` シート `18就労継続支援(B・基本)` の決定3・4記載行の48列目と `r6-service-codes-2-pdf` pp.257-259を突合、(2) `r6-fee-notice` pp.146-152（第1条表・左欄）とpp.235-238（第2条表・左欄）の率・単位数を2方式で抽出して照合、(3) `current-fee-notice-html` の第14の該当項番号の条文値を照合、(4) `r8-service-codes-2-xlsx` の決定3記載行で同一コード・同一単位数を確認する。ハッシュ不一致時は値を使わず `docs/open-questions.md` に起票して停止する（ADR 0020）。
