# ADR 0027: 令和6年度 就労継続支援B型 基本報酬・サービスコード・地域区分単価の実値

## 結論

令和6年度改定（2024-04適用）の就労継続支援B型基本報酬135区分を、ハッシュ照合済み一次資料2点（`r6-fee-notice` と `r6-service-codes-2-xlsx`）から独立に抽出し、全135値の一致を確認して確定した。内訳はサービス費（Ⅰ）〜（Ⅲ）＝人員配置3種×定員5区分×平均工賃月額8区分の120行、参加評価型のサービス費（Ⅳ）〜（Ⅵ）＝人員配置3種×定員5区分の15行である。各行の6桁サービスコード（種類46＋項目4桁）と略称は、`r8-service-codes-2-xlsx`（令和8年6月施行分・Ｒ8改定対象外欄）でも同一コード・同一単位数であることを全行照合した。地域区分単価は `mhlw-unit-price-notice-observed-946c3d96`（厚生労働省告示第539号）から、就労継続支援＝10円×告示割合として8地域区分すべてを確定した。本ADRの値は、Phase 3-1のseed JSON（Task 3）とgolden case期待値（Task 6）の唯一の出典である。確定できなかった行はない。Ｒ8改定対象の新12区分（新設コード群）は本ADRのスコープ外であり、値の流用を禁止する。

## 背景

- ハード制約3（CLAUDE.md）は、単位数・加算・地域区分単価・サービスコードのハードコードを禁止し、一次資料から抽出しADRで確定した値だけをシードとして許可する。
- ADR 0020が請求マスタの出典（documentId・sha256）と版（`claim-master-r6-04` 〜 `claim-master-r8-06`）を確定済み。basic-rewardsの出典は `r6-fee-notice`・`r6-reward-structure`、service-codesの出典は `r6-service-codes-2-xlsx`（R6）と `r8-service-codes-2-xlsx`（R8-6月）、region-unit-pricesの出典は `mhlw-unit-price-notice-observed-946c3d96` と `r6-revision-overview` である。
- ADR 0025が請求算定の端数規則（丸め位置・適用順・固定小数契約）を確定済み。本ADRの手計算検証ケースはADR 0025の適用順に従う。
- 令和6年度改定は人員配置6:1の報酬体系（サービス費（Ⅰ）・（Ⅳ））を新設した（`r6-revision-overview` p.52「（３）就労継続支援Ｂ型」）。7.5:1はサービス費（Ⅱ）・（Ⅴ）、10:1はサービス費（Ⅲ）・（Ⅵ）に対応する。

## 選択肢

### A: 告示とサービスコード表Excelの二重照合で135行全て採録する（採用）

- 同一値を独立した2資料から抽出して突合するため、転記事故・読取り事故を機械的に検出できる。実施の結果、135/135で一致した。
- Task 3のseedがB型基本報酬の全行を必要とするため、再抽出の手戻りがない。

### B: 垂直スライス最小限（定員20人以下×7.5:1・10:1）だけ採録する

- 最小カバレッジ（1定員区分×全工賃区分×両人員配置＋地域単価）は満たすが、Task 3で残り95行の抽出・照合を再度行うことになる。二重照合が全行で成立した以上、分割する理由がない。不採用。

### C: サービスコード表Excel単独から抽出する

- 告示との突合がなく、Excel側の合成単位数の誤り・読取り誤りを検出できない。不採用。

### 2. B型基本報酬とサービスコード（R6値・135行）

出典locatorの記法:

- 「告示頁」= `r6-fee-notice`（令和6年こども家庭庁・厚生労働省告示第3号PDF）の物理頁。同PDFは新旧対照表様式であり、値はすべて左欄（改正後）の「第14 就労継続支援Ｂ型」の1「就労継続支援Ｂ型サービス費（１日につき）」から読み取った。サービス費（Ⅰ）・（Ⅳ）は改正前欄が（新設）である。
- 「R6行」= `r6-service-codes-2-xlsx` のシート `18就労継続支援(B・基本)` の行番号。「R8行」= `r8-service-codes-2-xlsx` の同名シートの行番号。単位数はいずれも48列目「合成単位数」。サービスコードは種類コード46＋項目4桁。算定単位は「1日につき」。
- 全135行で、告示の単位数・R6 Excelの合成単位数・R8-6月 Excelの合成単位数の三者が一致した。R8-6月表の略称は（一）〜（六）区分だけ「改定外」入り（例: 就継ＢⅠ１１ → 就継ＢⅠ１改定外１）で、（七）（八）と参加評価型は同一略称である。

### 4. 手計算検証ケース（golden case期待値）

端数規則はADR 0025に従う: 月次給付単位数は整数合算（丸めなし）、総費用額＝給付単位数×地域単価の円未満切捨て、1割相当額＝総費用額×10/100の円未満切捨て、給付費＝総費用額−決定利用者負担額（90%乗算はしない）。3ケースとも、法31条特例は不適用（`NotApplicable`）、受給者証上限・制度上限は1割相当額以上、上限額管理の対象外、加算・減算なし（基本報酬のみ）という入力を前提とする。この前提では決定利用者負担額＝1割相当額である。

#### ケース1: cap-20-or-less × band-20000-25000 × staff-7.5-1 × 22日 × region-grade-2

- サービスコード 462049（就継ＢⅡ１５）＝637単位/日
- 月次給付単位数: 637 × 22 = 14,014単位
- 総費用額: 14,014 × 10.91円 = 152,892.74円 → 円未満切捨て → **152,892円**
- 1割相当額: 152,892 × 10/100 = 15,289.2円 → 円未満切捨て → 15,289円
- 決定利用者負担額: **15,289円**（前提により1割相当額）
- 給付費: 152,892 − 15,289 = **137,603円**

#### ケース2: cap-20-or-less × band-under-10000 × staff-10-1 × 20日 × region-other

- サービスコード 462883（就継ＢⅢ１８）＝490単位/日

- 総費用額: 9,800 × 10.00円 = 98,000.00円 → 円未満切捨て → **98,000円**
- 1割相当額: 98,000 × 10/100 = 9,800.0円 → 円未満切捨て → 9,800円
- 決定利用者負担額: **9,800円**
